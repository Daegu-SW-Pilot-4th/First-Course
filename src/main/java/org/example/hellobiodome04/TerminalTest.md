# 🧑‍💻 HelloBiodome04 터미널 테스트 가이드

---

## 💻 실행 환경 설정

`java_first_course` 디렉토리에서 아래 명령어를 실행한다.

**1단계 — 출력 디렉토리 생성 및 컴파일**
```bash
# Windows (PowerShell / CMD)
mkdir out
javac -d out src\main\java\org\example\hellobiodome04\HelloBiodome04.java

# macOS / Linux
mkdir out
javac -d out src/main/java/org/example/hellobiodome04/HelloBiodome04.java
```

**2단계 — 실행 기본 형태**
```bash
java -cp out org.example.hellobiodome04.HelloBiodome04 <온도> <습도> <산소농도>
```

---

## 🎯 테스트 케이스

편의를 위해 이하 커맨드에서 클래스패스 부분은 `[CP]`로 표기한다.

```
[CP] = -cp out org.example.hellobiodome04.HelloBiodome04
```

### TC-01 기본 안정 입력

```bash

```

| 항목 | 내용 |
|---|---|


### TC-09 습도만 정상 범위 이탈 (하한 미만)

```bash
java [CP] 25.5 35 21.0
```

| 항목 | 내용 |
|---|---|
| 예상 출력 | `-> 습도값이 정상 범위를 벗어났습니다. 확인이 필요합니다.` |
| 확인 사항 | 습도(40 초과 60 미만)만 벗어난 경우 정확히 습도값으로 안내한다 (온도/습도/산소농도 판별 로직 회귀 테스트) |

> **회귀 테스트:** 초기 구현은 온도 범위 판단 메서드에 `humidity`를 잘못 전달하는 오류가 있어, 습도·산소농도 이탈 시에도 항상 "온도값이 정상 범위를 벗어났습니다"로 잘못 출력되는 버그가 있었다. 이 케이스는 해당 버그가 재발하지 않는지 확인하기 위한 테스트다.

### TC-11 Requirements.md 기본 예시 재현 (문서 확인용)

```bash
java [CP] 25.5 65.0 21.0
```

| 항목 | 내용 |
|---|---|

> **문서 확인:** `Requirements.md`의 "결과 예시"에는 이 입력이 `생명의 나무는 안정적인 상태입니다 :)`로 안내되어 있으나, 습도 안정 범위(40% 초과 60% 미만) 규정과 모순된다(65.0은 범위 밖). 이는 문제3의 예시를 그대로 재사용하며 생긴 문서 자체의 오류로 보이며, 구현은 안정 범위 규정을 그대로 따랐다.

### TC-12 온도 경계값 (하한 포함 / 상한 제외)

```bash
java [CP] 10 55 21
java [CP] 27.5 55 21
```

| 입력 | 예상 출력 | 확인 사항 |
|---|---|---|
| `10 55 21` | `-> 생명의 나무는 안정적인 상태입니다. 건강지수는 3.20입니다.` | 온도 하한 10은 "이상" 조건이라 포함되어 안정으로 판단된다 |
| `27.5 55 21` | `-> 온도값이 정상 범위를 벗어났습니다. 확인이 필요합니다.` | 온도 상한 27.5는 "미만" 조건이라 제외되어 이탈로 판단된다 |

### TC-13 습도 경계값 (양쪽 모두 제외)

```bash
java [CP] 25.5 40 21.0
java [CP] 23.1 60.0 23.0
```

| 입력 | 예상 출력 | 확인 사항 |
|---|---|---|
| `25.5 40 21.0` | `-> 습도값이 정상 범위를 벗어났습니다. 확인이 필요합니다.` | 습도 하한 40은 "초과" 조건이라 제외되어 이탈로 판단된다 |
| `23.1 60.0 23.0` | `-> 습도값이 정상 범위를 벗어났습니다. 확인이 필요합니다.` | 습도 상한 60은 "미만" 조건이라 제외되어 이탈로 판단된다 |

> **문서 확인:** `Requirements.md`의 보너스 과제 예시는 이 입력(`23.1 60.0 23.0`)을 안정 상태(건강지수 9.23)로 안내하지만, 습도 안정 범위 규정("60% 미만")을 문자 그대로 적용하면 60.0은 범위 밖이다. 구현은 규정 텍스트를 그대로 따르기로 했다.
