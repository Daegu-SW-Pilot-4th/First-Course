## 💻 실행 환경 설정

`java_first_course` 디렉토리에서 아래 명령어를 실행한다.

**1단계 — 출력 디렉토리 생성 및 컴파일**
```bash
# Windows (PowerShell / CMD)
mkdir out
javac -d out src\main\java\org\example\hellobiodome07\HelloBiodome07.java

# macOS / Linux
mkdir out
javac -d out src/main/java/org/example/hellobiodome07/HelloBiodome07.java
```

**2단계 — 실행 기본 형태**
```bash
java -cp out org.example.hellobiodome07.HelloBiodome07 <DNA 염기서열...>
```

## 🎯 테스트 케이스

편의를 위해 이하 커맨드에서 클래스패스 부분은 `[CP]`로 표기한다.

```
[CP] = -cp out org.example.hellobiodome07.HelloBiodome07
```

### TC-04 공백 포함 입력 — 보너스 동작 확인 (Requirements 기본 예시와 차이)

```bash
java [CP] EEE EECCCCYYY
```

| 항목 | 내용 |
|---|---|
| 예상 출력 | `-> E3 E2C4Y3` |
| 확인 사항 | 공백을 사이에 두고 같은 문자가 이어져도, 보너스 동작(공백 유지)에서는 경계 너머로 병합되지 않는다 |

> **문서 확인:** `Requirements.md`의 기본 결과 예시는 이 입력을 공백을 완전히 제거하고 병합한 `E5C4Y3`으로 안내한다. 이 구현은 보너스 과제(공백 1개 유지, 경계 미병합)를 선택했기 때문에 실제 출력은 `E3 E2C4Y3`이다. 자세한 내용은 `README.md`를 참고한다.

### TC-06 빈 문자열 인자 (회귀 테스트)

```bash
java [CP] ""
```

| 항목 | 내용 |
|---|---|
| 예상 출력 | `-> 염기서열이 입력되지 않았습니다. 다시 입력해주세요` |
| 확인 사항 | 인자는 존재하지만 내용이 빈 문자열인 경우에도 동일하게 미입력으로 처리된다 |

> **회귀 테스트:** 초기 구현은 `args.length >= 1`만으로 입력 여부를 판단해, 이 경우 초기화되지 않은 문자(`\0`)가 그대로 출력되는 버그가 있었다. 정규화된 문자열이 실제로 비어 있는지를 기준으로 판단하도록 수정해 해결했다.

### TC-12 대소문자 혼합 + 다중 인자 — CodeValidationGuideline 예시 재현 (문서 확인용)

```bash
java [CP] CCJJhhJJJJJ EEEEEHHHH jjjCCCCC
```

| 항목 | 내용 |
|---|---|
| 예상 출력 | `-> C2J2H2J5 E5H4 J3C5` |
| 확인 사항 | 대소문자가 섞인 여러 인자를 대문자로 정규화하면서, 인자 사이 공백은 보너스 동작에 따라 결과에 남는다 |

> **문서 확인:** `CodeValidationGuideline.md`의 기본 확인 예시는 이 입력을 공백 없이 병합한 `C2J2H2J5E5H4J3C5`로 안내하지만, 이 구현은 보너스 과제(공백 유지)를 선택했기 때문에 공백이 남은 `C2J2H2J5 E5H4 J3C5`가 정확한 결과다. 자세한 내용은 `README.md`를 참고한다.
