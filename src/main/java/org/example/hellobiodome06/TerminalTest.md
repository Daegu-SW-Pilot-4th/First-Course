## 💻 실행 환경 설정

`java_first_course` 디렉토리에서 아래 명령어를 실행한다.

**1단계 — 출력 디렉토리 생성 및 컴파일**
```bash
# Windows (PowerShell / CMD)
mkdir out
javac -d out src\main\java\org\example\hellobiodome06\HelloBiodome06.java

# macOS / Linux
mkdir out
javac -d out src/main/java/org/example/hellobiodome06/HelloBiodome06.java
```

**2단계 — 실행 기본 형태**
```bash
java -cp out org.example.hellobiodome06.HelloBiodome06 <유전자코드1> <유전자코드2>
```

---

## 🎯 테스트 케이스

편의를 위해 이하 커맨드에서 클래스패스 부분은 `[CP]`로 표기한다.

```
[CP] = -cp out org.example.hellobiodome06.HelloBiodome06
```

### TC-01 동일한 유전자 코드

```bash
java [CP] sfd215j sfd215j
```

| 항목 | 내용 |
|---|---|
| 예상 출력 | `-> 동일한 유전자 코드입니다.`<br>`-> 포함되지 않습니다.` |

### TC-08 중간에 포함되는 경우 (회귀 테스트)

```bash
java [CP] dna12 xxdna12yy
```

| 항목 | 내용 |
|---|---|
| 예상 출력 | `-> 일치하지 않습니다.`<br>`-> 부분적으로 포함됩니다.` |
| 확인 사항 | gene1이 gene2의 맨 앞이 아니라 중간에 위치해도 정확히 포함으로 판단된다 |

> **회귀 테스트:** 초기 구현은 두 문자열을 인덱스 0부터만 나란히 비교해, gene1이 gene2의 접두사일 때만 포함으로 판단하는 오류가 있었다. 이 케이스는 gene2의 중간 위치에 있는 부분 문자열도 정확히 탐지하는지 확인하기 위한 테스트다.

### TC-14 Requirements.md 기본 예시 재현 (문서 확인용)

```bash
java [CP] sfd215j asfd215j
```

| 항목 | 내용 |
|---|---|
| 예상 출력 | `-> 일치하지 않습니다.`<br>`-> 부분적으로 포함됩니다.` |
| 확인 사항 | 두 문자열이 실제로는 다르다는 것과, 첫 번째 코드가 두 번째 코드에 부분적으로 포함된다는 것을 함께 확인한다 |

> **문서 확인:** `Requirements.md`의 결과 예시는 이 입력을 `"동일한 유전자 코드입니다."`로 안내하지만, `asfd215j`는 `sfd215j` 앞에 `a`가 하나 더 붙은 다른 문자열이라 실제로는 동일하지 않다. 다만 `sfd215j`가 `asfd215j`의 1번 인덱스부터 정확히 일치하므로, 공교롭게도 보너스(부분 포함) 기능의 유효한 예시가 된다.
