# 🧑‍💻 HelloBiodome03 터미널 테스트 가이드

---

## 💻 실행 환경 설정

`java_first_course` 디렉토리에서 아래 명령어를 실행한다.

**1단계 — 출력 디렉토리 생성 및 컴파일**
```bash
# Windows (PowerShell / CMD)
mkdir out
javac -d out src\main\java\org\example\hellobiodome03\HelloBiodome03.java

# macOS / Linux
mkdir out
javac -d out src/main/java/org/example/hellobiodome03/HelloBiodome03.java
```

**2단계 — 실행 기본 형태**
```bash
java -cp out org.example.hellobiodome03.HelloBiodome03 <온도> <습도> <산소농도>
```

---

## 🎯 테스트 케이스

편의를 위해 이하 커맨드에서 클래스패스 부분은 `[CP]`로 표기한다.

```
[CP] = -cp out org.example.hellobiodome03.HelloBiodome03
```

### TC-08 습도 음수 입력 (회귀 테스트)

```bash
java [CP] 25.5 -10.0 21.0
```

| 항목 | 내용 |
|---|---|
| 예상 출력 | `-> 입력된 값이 올바르지 않습니다. [온도][습도][산소농도] 순서대로 숫자 값을 입력해주세요.` |
| 확인 사항 | `calcSquareRoot()`가 음수 입력 시 예외를 던지고, `calcHealthScale()`이 이를 잡아 안내 문구 출력 후 종료된다 |

> **회귀 테스트:** 수정 전 코드는 음수 습도 입력 시 `calcSquareRoot()`가 `Double.NaN`을 반환해 예외 없이 `H = NaN`이 그대로 출력되는 버그가 있었다. 이 케이스는 해당 버그가 재발하지 않는지 확인하기 위한 테스트다.
